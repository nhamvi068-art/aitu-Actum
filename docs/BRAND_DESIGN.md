# Opentu 品牌设计方案

## 1. 品牌升级结论

Opentu 不再定义为“AI 图片视频创作工具 + 一体化白板”，而是升级为：

> **以画布为核心工作区的 AI 应用平台**

这次升级的重点不是简单扩功能，而是把原本散落在“白板”“AI 生成”“工具箱”“知识库”“工作流”中的能力，重新组织成一个更高层次的平台品牌。

## 2. 为什么要从工具升级为平台

旧叙事的问题：

- 太依赖“白板工具”心智，限制了产品延展
- AI 图片、视频、工作流、知识库、Agent 等能力缺少统一世界观
- 品牌看起来像“白板上叠了一些 AI 功能”，而不是一个完整系统

新叙事的目标：

- 对外统一成 `Opentu`
- 对内统一成“工作区 + 应用层 + 运行层 + 开放层”
- 让用户理解 Opentu 承载的不只是创作，而是 AI 应用协作与交付

## 3. 品牌核心结构

### 3.1 工作区层

画布是 Opentu 的核心工作区底座。

它承担的不是传统白板工具角色，而是：

- AI 应用的统一承载界面
- 媒体与结构化内容的编辑空间
- 工作流结果的可视化落点
- 多任务、多素材、多上下文之间的组织中心

### 3.2 应用层

Opentu 的应用层由一组场景能力组成：

- 图片生成
- 视频生成
- 灵感板
- 思维导图
- 流程图
- 知识库
- 工具箱
- Agent / Skill

这些能力不再作为并列的“附加功能”，而是 Opentu 平台中的应用模块。

### 3.3 运行层

平台能力依赖统一运行层：

- 多模型路由
- 任务队列
- 工作流执行
- 素材库与缓存
- 同步与备份恢复

品牌叙事中要明确：Opentu 不只是界面，更有完整运行底座。

### 3.4 开放层

Opentu 的平台属性还来自：

- 插件化扩展
- 开源
- 私有部署
- 可演化的工具与能力注册机制

## 4. 品牌主张

### 一句话

`Opentu 是一个以画布为核心工作区的 AI 应用平台。`

### 短版

`在同一工作区中连接模型、工具、知识与工作流。`

### 长版

`Opentu 将图片、视频、图形编辑、知识沉淀、任务执行、Agent 协作与工具扩展组织到同一个画布工作区中，让 AI 应用的生成、编辑、分析、整理与交付可以在一个平台内完成。`

## 5. 品牌个性

- 开放：不是封闭单点工具，而是可扩展平台
- 组织力：强调工作区、结构与协作，而不是功能堆叠
- 智能：AI 不是装饰，而是平台运行核心
- 直接：减少空泛“创意感”，突出可执行与可落地

## 6. 目标用户

### 创作者

需要在同一空间里完成素材生成、编辑、整理与输出。

### 团队

需要把模型、知识、工作流与内容资产组织在统一工作区中。

### 开发者与高级用户

重视插件化、可扩展、可部署、可接入的开放平台能力。

## 7. 文案迁移规则

### 允许保留

- `Opentu（开图）`
- `画布工作区`
- `思维导图 / 流程图 / 灵感板 / 工具箱`
- `开源、可扩展、可部署`

### 必须替换

- `白板工具`
- `开源白板应用`
- `AI 图片视频创作工具 + 一体化白板`
- `whiteboard tool` 作为英文一级定位

### 替换方向

- `白板` → `画布工作区`
- `工具` → `平台能力` 或 `应用模块`
- `功能集合` → `平台结构`

## 8. 视觉方向

视觉上不再围绕“单一创作工具”做隐喻，而围绕“平台入口、工作区、节点连接、运行流转”组织。

保留当前橙金 + 蓝紫 + 玫红体系，但语义调整为：

- 蓝紫：工作区与平台
- 橙金：应用结果与执行反馈
- 玫红：连接、激活、创意动势

## 9. 实施优先级

### 第一阶段

- 统一 README 与 docs 入口文案
- 统一用户手册首页
- 统一站点 SEO 与结构化描述

### 第二阶段

- 清理应用内设置页、调试页、同步描述中的旧品牌语句
- 清理中英文大小写与拼写混用

### 第三阶段

- 根据新平台定位更新官网、分享图、Logo 应用与发布物料
