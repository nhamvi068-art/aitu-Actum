# PPT 公共提示词历史经验

## 背景

PPT 大纲视图的公共提示词会影响整套幻灯片风格，但此前没有独立历史入口。用户需要复用同一套风格描述时，只能手动复制，且容易和图片、Agent 等提示词历史混在一起。

## 经验

- 公共提示词应作为独立提示词类别保存，使用 `ppt-common`，避免污染 `image` / `agent` 历史。
- 优先复用现有 `PromptType`、`usePromptHistory`、`PromptListPanel`，不要给 PPT frame 元数据新增历史字段。
- 公共提示词历史只记录用户真实输入或优化后的内容，默认预设保持为空，减少列表噪声。
- 优化弹窗需要支持 `historyType` 覆盖，否则公共提示词优化会按生图类型写入 `image` 历史。
- 大纲视图只接入历史选择、置顶、删除等 UI，不改变 PPT 生成队列和 frame 持久化结构。

## 实现要点

- 在公共提示词标题右侧增加 `History` icon，点击后打开复用的历史列表面板。
- 公共提示词失焦保存、批量生成前保存时，统一写入 `ppt-common` 历史。
- 选择历史后同步更新公共提示词草稿，并沿用原有 `setFramePPTMeta` 写入所有页面。
- `PromptGenerationType` 和存储层同步扩展 `ppt-common`，并为该类别返回空默认预设。

## 注意事项

- 不要新增 PPT 专用历史表，避免和全局提示词历史出现双写。
- 不要把公共提示词写入图片历史，否则会干扰生图入口的提示词推荐。
- 不要在历史列表中常驻大内容预览；列表只保存短文本状态，交互层按需展开。
- 新增提示词类别时，类型定义、默认预设、排序/置顶存储要一起覆盖。
