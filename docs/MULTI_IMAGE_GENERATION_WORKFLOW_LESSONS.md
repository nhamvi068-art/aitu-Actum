# 多图生成工作流经验总结

更新日期：2026-04-30

## 背景

「多图生成」原本以连环画为主要入口，但实际承载的是多页图片规划、批量生成和导出流程。它需要覆盖故事分镜、教程步骤、产品手册、营销图文等更多场景，同时保持批量生成时的稳定性和低内存占用。

## 经验

### 1. 改展示名，不要轻易改工具 id

面向用户的工具名可以从「连环画生成」扩展为「多图生成」，但内部 `comic-creator` id、存储 key、任务元数据不应同步重命名。展示层改名能降低迁移风险，也避免历史记录、任务恢复和懒加载注册失效。

落地原则：

- 用户可见的 `name` / `description` 放宽到多场景表达。
- 内部 id、组件名、存储 key 保持稳定。
- 业务文案可扩展，数据结构不为一次改名付出迁移成本。

### 2. 提示词优化入口优先复用公共能力

公共提示词和单页提示词都需要优化入口，但不必新增 `comic.common` / `comic.page` 场景实体。现有 `PromptOptimizeButton` 与 `tool.image` 场景已经能覆盖图片生成提示词优化。

落地原则：

- 公共提示词使用普通润色，重点收敛整套图片的统一风格。
- 单页提示词允许结构化 JSON，保留更强的画面可控性。
- 优化 icon 放在标题行，避免占用 textarea 编辑区。
- 不新增场景实体，除非现有 `tool.image` 已无法表达业务差异。

### 3. 串行生成应跳过已有图，但不能丢失连续性

串行生成的价值在于后一页参考前一页图片。如果跳过已生成页时简单从待生成列表取上一项，会丢失真实上一页参考图。

更稳的做法：

- 批量串行时过滤已有 `imageUrl` 的页面，避免重复消耗生成额度。
- 单页重生不跳过，保留用户主动覆盖能力。
- 每次生成当前页前，从最新记录里按 `pageNumber - 1` 查找上一页图片。
- 被跳过的已生成页仍可作为后一页参考图。

### 4. 首尾页操作要隐藏无效动作

第一页不能上移、最后一页不能下移。相比禁用按钮，直接不渲染无效 icon 更清爽，也减少误点击和 hover 噪音。

落地原则：

- `index > 0` 才显示上移。
- `index < pages.length - 1` 才显示下移。
- 批量生成中仍禁用有效的排序按钮，避免任务执行期间页面顺序变动。

## 检查清单

- 工具展示名是否只改用户可见层？
- 是否避免新增不必要的优化场景实体？
- 串行批量是否跳过已有图？
- 单页重生是否仍能覆盖已有图？
- 跳过页是否仍能被后一页作为参考图？
- 列表中的无效操作 icon 是否被隐藏而不是仅禁用？
- 提示词优化按钮是否禁用生成中状态？
