CSP 策略配置

为了允许当前应用被其他域名通过iframe嵌入，需要配置以下HTTP响应头： 
Content-Security-Policy: frame-ancestors 'self' localhost:* 127.0.0.1:* *.localhost:* https://api.tu-zi.com
# 不再设置 X-Frame-Options，避免与 frame-ancestors 冲突
当前策略允许嵌入的父域名:
• localhost (任意端口) - 本地开发环境
• 127.0.0.1 (任意端口) - 本地IP访问
• https://api.tu-zi.com - 生产环境授权域名
• 同域名 (self) - 同域内嵌入
使用说明:
1. 在 api.tu-zi.com 的网站中可以使用 <iframe src="你的应用URL"></iframe> 来嵌入此应用
2. 其他未授权域名将收到 "Refused to frame" 错误，无法嵌入
3. 配置已添加到 Vite 开发服务器、Netlify _headers 文件和 netlify.toml 中
4. Netlify 部署后会自动应用这些CSP策略
验证部署:
部署后可以通过浏览器开发者工具的 Network 标签查看响应头，确认包含：
Content-Security-Policy: frame-ancestors 'self' localhost:* 127.0.0.1:* *.localhost:* https://api.tu-zi.com
