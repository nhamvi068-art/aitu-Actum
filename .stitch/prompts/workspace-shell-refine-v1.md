# Workspace Shell Refinement V1

Refine the current `opentu` workspace shell so it matches the existing project code structure more closely.

## Match The Current Product Shell

- This is an editor shell, not a website or landing page
- Do not use a top navigation bar
- Do not make the canvas look like a dashboard with large sample cards
- Keep the canvas as a mostly open whiteboard workspace with only light content hints

## Required Structure

- Large central whiteboard canvas remains dominant
- Compact fixed vertical toolbar attached to the left edge
- Floating AI input bar at the bottom center
- Compact zoom and minimap controls at the top right area
- Small update hint can stay subtle, but not as a full top header
- Leave visual room for left and right drawers, command palette, search, and dialogs

## Closer To Code

- Left toolbar should feel denser and more tool-like
- AI input bar should feel like the current orange-accent composition bar, not a generic search field
- View navigation should sit near the upper-right corner, not the bottom-right corner
- Overall composition should feel like a real whiteboard editor already in use

## Avoid

- top website navigation
- hero section composition
- oversized demo cards on the canvas
- dashboard homepage feeling
- mobile layout patterns
